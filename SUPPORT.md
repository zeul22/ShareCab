# Support

This repository is for source code, development, documentation, and security
coordination. It is not an emergency support channel.

## Development Help

Use GitHub issues for:

- Reproducible bugs.
- Documentation gaps.
- Local setup problems.
- Focused feature proposals.

Use pull requests for small fixes with a clear explanation and verification.

## Security Reports

Do not open public issues for vulnerabilities. Follow [SECURITY.md](./SECURITY.md).

## Conduct Reports

Follow [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md).
